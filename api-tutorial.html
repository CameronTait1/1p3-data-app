1. How to Source Data

Data can be obtained from various sources including public APIs and datasets. Here are a few recommended sources:

  • OpenWeatherMap: Provides weather data including current conditions and forecasts. OpenWeatherMap API
  • CoinGecko: Offers cryptocurrency data such as prices, market cap, and volume. CoinGecko API
  • COVID-19 Data: Global COVID-19 statistics and information. COVID-19 API

Sign up for an API key where required and review the API documentation for usage guidelines.

2. Understanding Data Structure

APIs typically return data in JSON format. Here’s how to interpret common data structures:

  • Weather Data Example (OpenWeatherMap): 
    {
    "weather": [
        {
            "description": "clear sky"
        }
    ],
    "main": {
        "temp": 22.5,
        "humidity": 60,
        "pressure": 1012
    }
}
  • Cryptocurrency Data Example (CoinGecko): 
    {
    "bitcoin": {
        "usd": 30000
    },
    "ethereum": {
        "usd": 2000
    }
}

Identify key-value pairs and nested objects to extract relevant information for your visualization.

3. Making API Requests

To access API data, you make HTTP requests using methods like GET. Here’s an example using JavaScript:


fetch('https://api.openweathermap.org/data/2.5/weather?q=London&units=metric&appid=YOUR_API_KEY')
    .then(response => response.json())
    .then(data => {
        console.log(data);
    })
    .catch(error => {
        console.error('Error fetching weather data:', error);
    });

The example above uses YOUR_API_KEY as a placeholder for where the api key should be entered. Replace this with your actual API key. Make sure to check for any errors and validate the response structure.
